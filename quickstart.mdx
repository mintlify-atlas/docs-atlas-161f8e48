## Get started in three steps

This guide will help you set up fastlane for your iOS or Android project and run your first automated lane.

<Note>
  Make sure you've [installed fastlane](/installation) before continuing.
</Note>

### Step 1: Initialize fastlane

Navigate to your project directory and run the setup command:

<CodeGroup>

```bash iOS
cd your-ios-project
fastlane init
```

```bash Android
cd your-android-project
fastlane init
```

```bash With Bundler
cd your-project
bundle exec fastlane init
```

</CodeGroup>

fastlane will ask you what you want to automate. Choose from:

1. **Automate screenshots** - Set up snapshot to capture screenshots
2. **Automate beta distribution** - Set up TestFlight or Google Play beta deployment
3. **Automate App Store distribution** - Set up automatic releases
4. **Manual setup** - Create a basic Fastfile to customize yourself

<Tip>
  If you're not sure, choose **Manual setup** to start with a basic configuration you can customize later.
</Tip>

### Step 2: Explore your Fastfile

After initialization, fastlane creates a `fastlane` directory with:

- **Fastfile** - Contains your lanes (automation workflows)
- **Appfile** - Stores app identifier and Apple ID (iOS) or package name (Android)

Open `fastlane/Fastfile` to see your lanes. Here's an example iOS Fastfile:

```ruby fastlane/Fastfile
default_platform(:ios)

platform :ios do
  desc "Run tests"
  lane :test do
    scan
  end

  desc "Build app"
  lane :build do
    gym
  end

  desc "Deploy to TestFlight"
  lane :beta do
    build_app
    upload_to_testflight
  end
end
```

Each `lane` is an automation workflow you can run. The `desc` describes what the lane does.

### Step 3: Run your first lane

Run a lane to see fastlane in action:

<CodeGroup>

```bash Run a specific lane
fastlane test
```

```bash With Bundler
bundle exec fastlane test
```

```bash Specify platform
fastlane ios test
```

</CodeGroup>

fastlane will execute all actions in that lane sequentially.

<Tip>
  Run `fastlane lanes` to see all available lanes in your Fastfile.
</Tip>

## Common lanes to add

Here are some useful lanes to add to your Fastfile:

<AccordionGroup>
  <Accordion title="iOS: Deploy to TestFlight" icon="plane">
    ```ruby fastlane/Fastfile
    lane :beta do
      increment_build_number
      build_app
      upload_to_testflight
    end
    ```

    Run with: `fastlane beta`
  </Accordion>

  <Accordion title="iOS: Deploy to App Store" icon="app-store-ios">
    ```ruby fastlane/Fastfile
    lane :release do
      capture_screenshots
      build_app
      upload_to_app_store
    end
    ```

    Run with: `fastlane release`
  </Accordion>

  <Accordion title="Android: Deploy to Google Play beta" icon="google-play">
    ```ruby fastlane/Fastfile
    lane :beta do
      gradle(task: "assemble", build_type: "Release")
      upload_to_play_store(track: "beta")
    end
    ```

    Run with: `fastlane beta`
  </Accordion>

  <Accordion title="Run tests and generate coverage" icon="magnifying-glass">
    ```ruby fastlane/Fastfile
    lane :test do
      scan(
        scheme: "YourApp",
        code_coverage: true
      )
    end
    ```

    Run with: `fastlane test`
  </Accordion>
</AccordionGroup>

## Using environment variables

Store sensitive data like API keys in a `.env` file instead of hardcoding them:

```bash fastlane/.env
APP_IDENTIFIER=com.example.app
APPLE_ID=your@email.com
```

Access them in your Fastfile:

```ruby fastlane/Fastfile
lane :deploy do
  app_identifier = ENV["APP_IDENTIFIER"]
  # your code here
end
```

<Warning>
  Add `.env` to your `.gitignore` to avoid committing sensitive credentials.
</Warning>

## Using with CI/CD

fastlane works seamlessly with continuous integration services:

<CodeGroup>

```yaml GitHub Actions
name: Deploy
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Ruby
        uses: ruby/setup-ruby@v1
        with:
          ruby-version: 3.2
          bundler-cache: true
      - name: Run fastlane
        run: bundle exec fastlane beta
```

```yaml CircleCI
version: 2.1
jobs:
  deploy:
    macos:
      xcode: 15.0.0
    steps:
      - checkout
      - run: bundle install
      - run: bundle exec fastlane beta
```

</CodeGroup>
