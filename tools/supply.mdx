## Overview

supply is fastlane's tool for uploading Android apps and their metadata to the Google Play Store. It handles APKs, AABs, screenshots, and store listings.

<Info>
supply is the easiest way to automate your Google Play Store releases.
</Info>

## Installation

supply is part of fastlane. Install it with:

```bash
fastlane supply
```

## Quick Start

<Steps>
  <Step title="Set up Google Play credentials">
    Create a service account and download the JSON key file. See [Google Play setup guide](https://docs.fastlane.tools/getting-started/android/setup/).
  </Step>
  
  <Step title="Initialize supply">
    ```bash
    fastlane supply init
    ```
    
    This downloads your existing metadata into a `metadata/android` folder.
  </Step>
  
  <Step title="Upload to Play Store">
    ```bash
    fastlane supply
    ```
  </Step>
</Steps>

## Authentication

### Service Account JSON

Use a Google Play service account:

```ruby
supply(
  json_key: "./google-play-key.json",
  package_name: "com.example.app"
)
```

Or use JSON data directly:

```ruby
supply(
  json_key_data: ENV["GOOGLE_PLAY_JSON_KEY_DATA"],
  package_name: "com.example.app"
)
```

<ParamField path="aab_paths" type="array">

  ```ruby
  supply(aab_paths: [
  ```

<ParamField path="track" type="string" default="production">

<ParamField path="skip_upload_changelogs" type="boolean" default="false">

<ParamField path="skip_upload_images" type="boolean" default="false">
  Whether to skip uploading images.

## Using with fastlane

### Basic Upload

```ruby
lane :deploy do
  gradle(task: "bundle")
  supply
end
```

### Upload to Beta Track

```ruby
lane :beta do
  gradle(
    task: "bundle",
    build_type: "Release"
  )
  
  supply(
    track: "beta",
    aab: "./app/build/outputs/bundle/release/app-release.aab"
  )
end
```

### Production Release with Rollout

```ruby
lane :production do
  gradle(task: "bundle")
  
  supply(
    track: "production",
    rollout: 0.1,  # Start with 10%
    aab: "./app/build/outputs/bundle/release/app-release.aab"
  )
end
```

### Promote Beta to Production

```ruby
lane :promote_to_production do
  supply(
    track: "beta",
    track_promote_to: "production",
    skip_upload_apk: true,
    skip_upload_aab: true
  )
end
```

### Upload Metadata Only

```ruby
lane :update_metadata do
  supply(
    skip_upload_apk: true,
    skip_upload_aab: true,
    skip_upload_screenshots: true
  )
end
```

## Metadata Management

### Folder Structure

supply organizes metadata by locale:

```
metadata/android/
├── en-US/
│   ├── title.txt
│   ├── short_description.txt
│   ├── full_description.txt
│   ├── video.txt
│   ├── images/
│   │   ├── featureGraphic.png
│   │   ├── icon.png
│   │   ├── phoneScreenshots/
│   │   ├── sevenInchScreenshots/
│   │   ├── tenInchScreenshots/
│   │   └── tvScreenshots/
│   └── changelogs/
│       └── 100.txt
└── de-DE/
    └── ...
```

### Download Existing Metadata

```bash
fastlane supply init
```

### Metadata Files

- `title.txt` - App title (50 characters max)
- `short_description.txt` - Short description (80 characters max)
- `full_description.txt` - Full description (4000 characters max)
- `video.txt` - YouTube video URL

### Changelogs

Create changelog files named by version code:

```
metadata/android/en-US/changelogs/
├── 100.txt
├── 101.txt
└── 102.txt
```

## Supplyfile

Create a `Supplyfile` for persistent configuration:

```ruby
# Supplyfile

json_key "./google-play-key.json"
package_name "com.example.app"

track "production"
release_status "completed"

metadata_path "./metadata/android"

skip_upload_images false
skip_upload_screenshots false
skip_upload_metadata false
```

## Tips & Best Practices

<Tip>
**Use AAB over APK**: Android App Bundles reduce download size and are required for new apps.
</Tip>

<Warning>
**Protect your service account**: Keep the JSON key file secure and never commit it to version control.
</Warning>

<Tip>
**Start with staged rollout**: Use `rollout: 0.1` to release to 10% of users first, then gradually increase.
</Tip>

<Check>
**Upload ProGuard mapping**: Include mapping files for better crash reports in the Play Console.
</Check>

## Troubleshooting

### Authentication failed

Verify your service account has the correct permissions in the Google Play Console.

### Package name mismatch

Ensure the package name matches your app in the Play Console:

```ruby
supply(package_name: "com.example.app")
```

### Track not found

Make sure the track exists. Valid tracks: `production`, `beta`, `alpha`, `internal`.
