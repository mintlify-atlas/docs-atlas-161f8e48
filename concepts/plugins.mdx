Plugins allow you to extend fastlane with custom actions that can be shared across projects and with the community. The plugin system makes it easy to package, distribute, and manage custom functionality beyond fastlane's built-in actions.

## What are Plugins?

A fastlane plugin is a Ruby gem that contains one or more custom actions. Plugins follow the naming convention `fastlane-plugin-[name]` and are distributed via RubyGems.org or other gem sources.

<Note>
Plugins are the recommended way to share custom actions with others or reuse them across multiple projects, rather than copying action files between projects.
</Note>

## Finding Plugins

fastlane provides built-in commands to discover available plugins:

```bash
# List all available plugins
fastlane search_plugins

# Search for specific plugins
fastlane search_plugins version
fastlane search_plugins firebase
fastlane search_plugins badge
```

You can also browse plugins at [fastlane.tools/plugins](https://fastlane.tools/plugins).

## Installing Plugins

### Add a Plugin

Use the `add_plugin` command to install a plugin:

```bash
fastlane add_plugin versioning
fastlane add_plugin badge
fastlane add_plugin firebase_app_distribution
```

This command:
1. Creates a `Pluginfile` in your `fastlane/` directory
2. Adds the plugin to the `Pluginfile`
3. Updates your `Gemfile` to load plugins
4. Runs `bundle install` to install the plugin

### Plugin Files

After adding a plugin, you'll have these files:

<Tabs>
  <Tab title="Pluginfile">
    ```ruby
    # Autogenerated by fastlane
    #
    # Ensure this file is checked in to source control!

    gem 'fastlane-plugin-versioning'
    gem 'fastlane-plugin-badge'
    gem 'fastlane-plugin-firebase_app_distribution'
    ```
    
    Located at: `fastlane/Pluginfile`
  </Tab>
  <Tab title="Gemfile">
    ```ruby
    source "https://rubygems.org"

    gem 'fastlane'

    plugins_path = File.join(File.dirname(__FILE__), 'fastlane', 'Pluginfile')
    eval_gemfile(plugins_path) if File.exist?(plugins_path)
    ```
    
    The `eval_gemfile` line loads your plugins.
  </Tab>
</Tabs>

<Warning>
**Always commit your Pluginfile**

Make sure to commit both `Pluginfile` and `Gemfile.lock` to version control so your team and CI can use the same plugin versions.
</Warning>

## Using Plugins

Once a plugin is installed, its actions are available in your Fastfile:

```ruby
lane :version_bump do
  # Actions from fastlane-plugin-versioning
  increment_version_number_in_plist(
    target: "MyApp",
    bump_type: "minor"
  )
  
  version = get_version_number_from_plist(
    target: "MyApp"
  )
  
  UI.message("New version: #{version}")
end

lane :icon do
  # Action from fastlane-plugin-badge
  add_badge(
    shield: "Version-#{get_version_number}-blue",
    dark: true
  )
end

lane :distribute do
  # Action from fastlane-plugin-firebase_app_distribution
  firebase_app_distribution(
    app: "1:1234567890:ios:abcdef1234567890",
    release_notes: "Bug fixes and improvements",
    groups: "testers, developers"
  )
end
```

## Managing Plugins

### Update Plugins

Update all plugins to their latest versions:

```bash
fastlane update_plugins
```

### Install Plugin Dependencies

After cloning a project or switching branches:

```bash
bundle install
```

### Remove a Plugin

Manually remove the plugin from your `Pluginfile`:

```ruby
# fastlane/Pluginfile
# Remove this line:
# gem 'fastlane-plugin-unwanted'
```

Then run:

```bash
bundle install
```

## Creating Your Own Plugin

### Generate a New Plugin

Use the built-in generator to create a new plugin:

```bash
fastlane new_plugin [plugin_name]
```

Example:

```bash
fastlane new_plugin my_company_tools
```

This creates a complete plugin structure:

```
fastlane-plugin-my_company_tools/
├── lib/
│   └── fastlane/
│       └── plugin/
│           └── my_company_tools/
│               ├── actions/
│               │   └── my_company_tools_action.rb
│               ├── helper/
│               │   └── my_company_tools_helper.rb
│               └── version.rb
├── spec/
│   └── my_company_tools_action_spec.rb
├── fastlane-plugin-my_company_tools.gemspec
├── Gemfile
├── Rakefile
└── README.md
```

### Plugin Structure

<Accordion title="Actions Directory">
  Contains your custom actions:
  
  ```ruby
  # lib/fastlane/plugin/my_plugin/actions/my_action.rb
  module Fastlane
    module Actions
      class MyActionAction < Action
        def self.run(params)
          UI.message("Running my custom action!")
          # Your action logic here
        end
        
        def self.description
          "My custom action description"
        end
        
        def self.available_options
          [
            FastlaneCore::ConfigItem.new(
              key: :parameter,
              description: "A parameter for my action",
              optional: false
            )
          ]
        end
        
        def self.is_supported?(platform)
          [:ios, :android].include?(platform)
        end
      end
    end
  end
  ```
</Accordion>

<Accordion title="Helper Directory">
  Contains helper modules for shared logic:
  
  ```ruby
  # lib/fastlane/plugin/my_plugin/helper/my_plugin_helper.rb
  module Fastlane
    module Helper
      class MyPluginHelper
        def self.show_message
          UI.message("Hello from the helper!")
        end
      end
    end
  end
  ```
  
  Use in your actions:
  
  ```ruby
  def self.run(params)
    Fastlane::Helper::MyPluginHelper.show_message
  end
  ```
</Accordion>

<Accordion title="Gemspec File">
  Defines your plugin's metadata and dependencies:
  
  ```ruby
  # fastlane-plugin-my_plugin.gemspec
  Gem::Specification.new do |spec|
    spec.name          = 'fastlane-plugin-my_plugin'
    spec.version       = '0.1.0'
    spec.author        = 'Your Name'
    spec.email         = 'your.email@example.com'
    
    spec.summary       = 'Short description of your plugin'
    spec.homepage      = 'https://github.com/yourname/fastlane-plugin-my_plugin'
    spec.license       = 'MIT'
    
    spec.files         = Dir["lib/**/*"] + %w(README.md LICENSE)
    spec.require_paths = ['lib']
    
    spec.add_dependency 'xcodeproj', '~> 1.0'
    
    spec.add_development_dependency 'bundler'
    spec.add_development_dependency 'rspec'
  end
  ```
</Accordion>

### Adding Actions to Your Plugin

Create additional actions in your plugin:

```bash
# From your plugin directory
fastlane new_action
```

Or manually create a new file:

```ruby
# lib/fastlane/plugin/my_plugin/actions/another_action.rb
module Fastlane
  module Actions
    class AnotherActionAction < Action
      def self.run(params)
        # Action implementation
      end
      
      def self.description
        "Another custom action"
      end
      
      def self.available_options
        []
      end
      
      def self.is_supported?(platform)
        true
      end
    end
  end
end
```

## Testing Your Plugin

### Write Tests

Plugins are generated with RSpec tests:

```ruby
# spec/my_action_spec.rb
require 'spec_helper'

describe Fastlane::Actions::MyActionAction do
  describe '#run' do
    it 'executes successfully' do
      expect(Fastlane::UI).to receive(:message)
        .with("Running my custom action!")
      
      Fastlane::Actions::MyActionAction.run(
        parameter: "test_value"
      )
    end
    
    it 'raises an error with invalid parameters' do
      expect do
        Fastlane::Actions::MyActionAction.run({})
      end.to raise_error("Missing required parameter")
    end
  end
end
```

### Run Tests

```bash
# From your plugin directory
bundle install
bundle exec rspec
```

### Test Locally

Test your plugin in a local project before publishing:

```ruby
# In your app's Gemfile
gem 'fastlane-plugin-my_plugin', path: '../fastlane-plugin-my_plugin'
```

Or in your `Pluginfile`:

```ruby
gem 'fastlane-plugin-my_plugin', path: '/absolute/path/to/plugin'
```

Then run:

```bash
bundle install
```

## Publishing Your Plugin

### Prepare for Release

1. **Update version number**:
   ```ruby
   # lib/fastlane/plugin/my_plugin/version.rb
   module Fastlane
     module MyPlugin
       VERSION = "0.1.0"
     end
   end
   ```

2. **Update README.md** with:
   - Clear description
   - Installation instructions
   - Usage examples
   - Available actions and parameters

3. **Test thoroughly**:
   ```bash
   bundle exec rspec
   ```

### Build the Gem

```bash
gem build fastlane-plugin-my_plugin.gemspec
```

This creates `fastlane-plugin-my_plugin-0.1.0.gem`.

### Publish to RubyGems

```bash
gem push fastlane-plugin-my_plugin-0.1.0.gem
```

<Note>
You'll need a RubyGems.org account. Sign up at [rubygems.org](https://rubygems.org).
</Note>

## Plugin Best Practices

<Accordion title="Follow Naming Conventions">
  - Plugin name: `fastlane-plugin-[name]`
  - Action file: `[name]_action.rb`
  - Action class: `[Name]Action`
  
  ```ruby
  # Good
  fastlane-plugin-versioning
  └── versioning_action.rb
      └── class VersioningAction
  
  # Bad
  fastlane-versioning-plugin
  └── version_action.rb
      └── class Version
  ```
</Accordion>

<Accordion title="Provide Comprehensive Documentation">
  ```ruby
  def self.description
    "Brief one-line description"
  end
  
  def self.details
    "Longer description explaining what the action does, "\
    "when to use it, and any important details."
  end
  
  def self.example_code
    [
      'my_action(param: "value")',
      'my_action(
        param: "value",
        another_param: true,
        optional_param: "optional"
      )'
    ]
  end
  ```
</Accordion>

<Accordion title="Handle Errors Gracefully">
  ```ruby
  def self.run(params)
    # Validate parameters
    UI.user_error!("Missing required file") unless File.exist?(params[:file])
    
    # Provide helpful error messages
    begin
      do_something_risky
    rescue => ex
      UI.user_error!("Failed to do something: #{ex.message}")
    end
    
    # Show success
    UI.success("Operation completed successfully!")
  end
  ```
</Accordion>

<Accordion title="Support Multiple Platforms">
  ```ruby
  def self.is_supported?(platform)
    # Support all platforms
    true
    
    # Or specific platforms
    [:ios, :mac].include?(platform)
  end
  
  def self.run(params)
    # Handle platform differences
    if [:ios, :mac].include?(Actions.lane_context[Actions::SharedValues::PLATFORM_NAME])
      # iOS/Mac specific logic
    elsif Actions.lane_context[Actions::SharedValues::PLATFORM_NAME] == :android
      # Android specific logic
    end
  end
  ```
</Accordion>

<Accordion title="Version Your Plugin Properly">
  Follow semantic versioning:
  - **Major** (1.0.0): Breaking changes
  - **Minor** (0.1.0): New features, backward compatible
  - **Patch** (0.0.1): Bug fixes
  
  ```ruby
  # Increment appropriately
  VERSION = "0.1.0"  # Initial release
  VERSION = "0.2.0"  # Added new action
  VERSION = "0.2.1"  # Fixed bug
  VERSION = "1.0.0"  # Stable API, no breaking changes
  VERSION = "2.0.0"  # Breaking API changes
  ```
</Accordion>

## Advanced Plugin Features

### Using Dependencies

Add gem dependencies in your gemspec:

```ruby
spec.add_dependency 'xcodeproj', '~> 1.0'
spec.add_dependency 'nokogiri', '~> 1.11'
spec.add_dependency 'aws-sdk-s3', '~> 1.0'
```

Use them in your actions:

```ruby
module Fastlane
  module Actions
    class MyActionAction < Action
      def self.run(params)
        require 'xcodeproj'
        require 'nokogiri'
        
        project = Xcodeproj::Project.open(params[:project_path])
        # Use the dependencies
      end
    end
  end
end
```

### Sharing Code Between Actions

Use helper modules:

```ruby
# lib/fastlane/plugin/my_plugin/helper/my_plugin_helper.rb
module Fastlane
  module Helper
    class MyPluginHelper
      def self.parse_version(version_string)
        version_string.match(/\d+\.\d+\.\d+/).to_s
      end
      
      def self.validate_file(path)
        UI.user_error!("File not found: #{path}") unless File.exist?(path)
      end
    end
  end
end

# Use in multiple actions
module Fastlane
  module Actions
    class FirstActionAction < Action
      def self.run(params)
        Helper::MyPluginHelper.validate_file(params[:file])
        version = Helper::MyPluginHelper.parse_version(params[:version])
      end
    end
    
    class SecondActionAction < Action
      def self.run(params)
        Helper::MyPluginHelper.validate_file(params[:input])
      end
    end
  end
end
```

## Troubleshooting

<Warning>
**Plugin Not Loading?**

If your plugin isn't loading:
1. Ensure you've run `bundle install`
2. Restart your terminal
3. Check that `Pluginfile` is in the `fastlane/` directory
4. Verify your `Gemfile` includes the `eval_gemfile` line
5. Run `bundle exec fastlane` instead of just `fastlane`
</Warning>

### Common Issues

```bash
# Error: No such file or directory @ rb_sysopen - Pluginfile
cd YourProject
fastlane add_plugin your_plugin  # Creates Pluginfile

# Error: Could not find gem 'fastlane-plugin-xyz'
bundle install  # Install plugin dependencies

# Error: Action 'my_action' not found
bundle exec fastlane actions  # Check if action is loaded
```

## Complete Plugin Example

Here's a complete, real-world plugin structure:

```ruby
# lib/fastlane/plugin/company_tools/actions/build_number_action.rb
module Fastlane
  module Actions
    class IncrementBuildNumberAction < Action
      def self.run(params)
        project = Xcodeproj::Project.open(params[:xcodeproj])
        build_number = params[:build_number]
        
        project.targets.each do |target|
          target.build_configurations.each do |config|
            config.build_settings['CURRENT_PROJECT_VERSION'] = build_number
          end
        end
        
        project.save
        
        Actions.lane_context[SharedValues::BUILD_NUMBER] = build_number
        UI.success("Build number set to: #{build_number}")
        
        build_number
      end
      
      def self.description
        "Set the build number for your Xcode project"
      end
      
      def self.details
        "Updates the CURRENT_PROJECT_VERSION in all configurations "\
        "and targets of your Xcode project."
      end
      
      def self.available_options
        [
          FastlaneCore::ConfigItem.new(
            key: :xcodeproj,
            env_name: "COMPANY_TOOLS_XCODEPROJ",
            description: "Path to your Xcode project",
            optional: false,
            verify_block: proc do |value|
              UI.user_error!("Could not find project at '#{value}'") unless File.exist?(value)
            end
          ),
          FastlaneCore::ConfigItem.new(
            key: :build_number,
            env_name: "COMPANY_TOOLS_BUILD_NUMBER",
            description: "The build number to set",
            optional: false,
            type: Integer
          )
        ]
      end
      
      def self.output
        [
          ['BUILD_NUMBER', 'The build number that was set']
        ]
      end
      
      def self.return_value
        "The build number that was set"
      end
      
      def self.authors
        ["Company Team"]
      end
      
      def self.is_supported?(platform)
        [:ios, :mac].include?(platform)
      end
      
      def self.example_code
        [
          'increment_build_number(
            xcodeproj: "MyApp.xcodeproj",
            build_number: 42
          )'
        ]
      end
      
      def self.category
        :project
      end
    end
  end
end
```
