This page provides a comprehensive list of all built-in fastlane actions, organized by category.

## Overview

Fastlane includes over 200 built-in actions covering every aspect of mobile app development and deployment. Each action is designed to handle a specific task and can be easily integrated into your lanes.

## How to Use This Reference

Each action listed includes:
- **Name**: The action name to use in your Fastfile
- **Category**: Functional category
- **Description**: Brief description of what the action does
- **Source**: Link to the source code

For detailed documentation on each action, visit [fastlane.tools/actions](https://docs.fastlane.tools/actions/).

## Deprecated Actions

These actions are deprecated and should not be used in new projects.

| Action | Description | Replacement |
|--------|-------------|-------------|
| `hockey` | Upload to HockeyApp | Use App Center plugin |
| `ipa` | Build using shenzhen | Use `build_app` |
| `notify` | macOS notification | Use `notification` |
| `xctool` | Use xctool | Use `xcodebuild` or `scan` |

## Finding More Information

For detailed documentation on each action:

1. Visit [docs.fastlane.tools/actions](https://docs.fastlane.tools/actions/)
2. Run `fastlane actions` to list all actions
3. Run `fastlane action [action_name]` for specific action details
4. Browse the [source code on GitHub](https://github.com/fastlane/fastlane/tree/master/fastlane/lib/fastlane/actions)

## Creating Custom Actions

If you need functionality not provided by built-in actions, you can create custom actions:

```bash
fastlane new_action
```

See the [Action API Reference](/api/action) for details on creating custom actions.
